# 러닝 피드 기능 논의 문서

## 1) 목표 요약
- 모임 생성 시작 단계에서 사용자에게 두 가지 선택지를 제공한다.
  - `같이 달리기`: 기존 모임 생성 플로우 그대로 진행
  - `러닝 피드`: Apple Fitness 러닝 기록 기반의 개인 공유 이미지 생성 플로우 진행

## 2) 현재까지 합의된 요구사항
1. 모임탭 최상단(헤더 위)에서 `러닝 피드` / `같이 달리기` 모드를 토글로 전환
2. `같이 달리기` 선택 시 기존 모임 생성 단계(현행 UX) 유지
3. `러닝 피드` 선택 시 Apple Fitness 러닝 기록 목록 표시
4. 각 러닝 기록마다 공유이미지 생성 버튼 제공

## 3) 논의가 필요한 항목 (확정 필요)

### A. "러닝 피드" 화면 진입 위치
- 후보 A1: 기존 "새 모임 만들기" 진입 직후 분기 화면 추가
- 후보 A2: 기존 Step 1 내부에 토글/탭 형태로 분기
- 후보 A3: 별도 화면으로 완전 분리
- 사용자 제안: 모임탭 최상단(헤더 위) 고정 토글(`러닝 피드` / `같이 달리기`)
- 현재 상태: 논의 중 (우선 확정 대상)

### B. Apple Fitness 기록 조회 범위
- 후보 B1: 최근 7일
- 후보 B2: 최근 30일
- 후보 B3: 무한 스크롤 (초기 30개 + 추가 로드)
- 확정: 최근 14일
- 현재 상태: 확정

### C. 러닝 기록 목록 정렬/표시 정보
- 정렬 기준 후보: 최신순(기본), 거리순, 시간순
- 카드에 표시할 정보 후보:
  - 운동 날짜/시간
  - 거리
  - 평균 페이스
  - 운동 시간
  - 칼로리
  - 경로 유무
- 확정: 최신순
- 현재 상태: 부분 확정 (표시 필드는 추가 합의 필요)

### D. 공유 이미지 생성 방식
- 후보 D1: 목록 아이템의 "이미지 생성" 클릭 시 즉시 생성
- 후보 D2: 상세 미리보기 화면에서 생성
- 후보 D3: 기존 `RunningShareModal` 재사용 + 선택 데이터 주입
- 확정: 후보 D1
- 현재 상태: 확정

### E. 장소(Place) 입력 처리
- 현재 공유 플로우는 Place 입력이 선행됨
- 확인 필요:
  - 러닝 피드에서도 Place 입력을 강제할지
  - Apple Fitness 기록의 위치 데이터가 있으면 자동 추정할지
  - 입력 필드 규칙(영문만 허용) 유지 여부
- 확정: 러닝 피드에서도 Place 입력 강제
- 현재 상태: 부분 확정 (자동추정/입력규칙은 추가 합의 필요)

### F. 저장/공유 동작 정의
- 후보 F1: 갤러리 저장만
- 후보 F2: 갤러리 저장 + 시스템 공유시트
- 후보 F3: 공유시트만 (저장 옵션은 별도)
- 확정: 후보 F1 (갤러리 저장만)
- 현재 상태: 확정

### G. 데이터 없을 때 처리
- Apple Fitness 권한 거부
- 기록 없음
- 경로 좌표 없음
- 조회 실패(네트워크/네이티브 오류)
- 현재 상태: 미확정

## 4) 제안하는 구현 방향 (초안)
1. 모임탭 최상단(헤더 위)에 모드 토글 추가 (`같이 달리기` / `러닝 피드`)
2. `같이 달리기` 탭은 기존 목록/생성 UX 유지
3. `러닝 피드` 탭은 최근 14일 Apple Fitness 러닝 기록을 최신순 표시
4. 각 기록 카드에서 "이미지 생성" 버튼 클릭 시 기존 공유 생성 로직 호출
5. Place는 생성 버튼 액션 전에 반드시 입력받아 검증
6. 결과물은 갤러리 저장만 수행

## 7) 질문 A 심화 논의 (모임탭 상단 토글)

### A-1. 토글 배치 제안
- 위치: 모임탭 화면 최상단, 헤더보다 위
- 형태: 2분할 세그먼트 컨트롤 (`같이 달리기` | `러닝 피드`)
- 상태 유지: 마지막 선택 모드를 로컬 저장(앱 재진입 시 복원)

### A-2. 탭별 화면 책임 분리 제안
- `같이 달리기`: 기존 일정/모임 카드 + 기존 새모임 생성 버튼
- `러닝 피드`: Apple Fitness 기록 리스트 + 각 아이템별 이미지 생성
- 장점: 기존 플로우 리스크 최소화, 사용자 인지 명확

### A-3. CTA(생성 버튼) 정책 제안
- `같이 달리기` 모드: 기존 "새 모임 만들기" 버튼 노출
- `러닝 피드` 모드: "새 모임 만들기" 버튼 숨김, 대신 각 기록 카드의 "이미지 생성" 사용
- 이유: 모드 목적과 행동 일치 (러닝 피드에서는 모임 생성이 핵심 행동이 아님)
- 확정: `러닝 피드`에서 새 모임(+) 버튼은 노출하지 않음

### A-4. 토글 전환 시 UX 합의 필요
- 스크롤 위치를 모드별로 기억할지 여부
- 전환 시 애니메이션(페이드/슬라이드) 적용 여부
- 토글 선택값 분석 이벤트 로깅 필요 여부

### A-5. 권장 1차 결정안 (MVP)
- 상단 고정 토글 적용
- 모드별 완전 분리 리스트 적용
- 모드별 CTA 분리(`같이`: 새 모임 생성 / `러닝 피드`: 기록별 이미지 생성)
- 전환 애니메이션은 최소화(기본), 성능 우선
