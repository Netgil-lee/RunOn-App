# 나 혼자 달리기 기능 논의 문서

## 1) 목표 요약
- 모임 생성 시작 단계에서 사용자에게 두 가지 선택지를 제공한다.
  - `같이 달리기`: 기존 모임 생성 플로우 그대로 진행
  - `혼자 달리기`: Apple Fitness 러닝 기록 기반의 개인 공유 이미지 생성 플로우 진행

## 2) 현재까지 합의된 요구사항
1. 모임생성 첫 화면에서 `혼자 달리기` / `같이 달리기` 버튼 노출
2. `같이 달리기` 선택 시 기존 모임 생성 단계(현행 UX) 유지
3. `혼자 달리기` 선택 시 Apple Fitness 러닝 기록 목록 표시
4. 각 러닝 기록마다 공유이미지 생성 버튼 제공

## 3) 논의가 필요한 항목 (확정 필요)

### A. "혼자 달리기" 화면 진입 위치
- 후보 A1: 기존 "새 모임 만들기" 진입 직후 분기 화면 추가
- 후보 A2: 기존 Step 1 내부에 토글/탭 형태로 분기
- 후보 A3: 별도 화면으로 완전 분리
- 현재 상태: 미확정

### B. Apple Fitness 기록 조회 범위
- 후보 B1: 최근 7일
- 후보 B2: 최근 30일
- 후보 B3: 무한 스크롤 (초기 30개 + 추가 로드)
- 현재 상태: 미확정

### C. 러닝 기록 목록 정렬/표시 정보
- 정렬 기준 후보: 최신순(기본), 거리순, 시간순
- 카드에 표시할 정보 후보:
  - 운동 날짜/시간
  - 거리
  - 평균 페이스
  - 운동 시간
  - 칼로리
  - 경로 유무
- 현재 상태: 미확정

### D. 공유 이미지 생성 방식
- 후보 D1: 목록 아이템의 "이미지 생성" 클릭 시 즉시 생성
- 후보 D2: 상세 미리보기 화면에서 생성
- 후보 D3: 기존 `RunningShareModal` 재사용 + 선택 데이터 주입
- 현재 상태: 미확정

### E. 장소(Place) 입력 처리
- 현재 공유 플로우는 Place 입력이 선행됨
- 확인 필요:
  - 혼자 달리기에서도 Place 입력을 강제할지
  - Apple Fitness 기록의 위치 데이터가 있으면 자동 추정할지
  - 입력 필드 규칙(영문만 허용) 유지 여부
- 현재 상태: 미확정

### F. 저장/공유 동작 정의
- 후보 F1: 갤러리 저장만
- 후보 F2: 갤러리 저장 + 시스템 공유시트
- 후보 F3: 공유시트만 (저장 옵션은 별도)
- 현재 상태: 미확정

### G. 데이터 없을 때 처리
- Apple Fitness 권한 거부
- 기록 없음
- 경로 좌표 없음
- 조회 실패(네트워크/네이티브 오류)
- 현재 상태: 미확정

## 4) 제안하는 구현 방향 (초안)
1. 새 모임 진입 직후 `모드 선택 단계` 추가 (`혼자 달리기` / `같이 달리기`)
2. `같이 달리기`는 기존 `RunningEventCreationFlow` 그대로 이동
3. `혼자 달리기`는 새 화면/모달 `SoloRunningHistoryScreen`으로 이동
4. `appleFitnessService`에 "최근 러닝 기록 목록" 조회 메서드 추가
5. 목록 아이템 선택 시 기존 공유 카드 생성 구조 재사용
6. 기존 `RunningShareModal`을 재사용 가능하도록 입력 모델 일반화
